Architecture &
Implementation Strategy

The TWA Claude OS uses Google Workspace as its backbone — shared Drive as the single source of truth, Workspace accounts to anchor Claude Pro seats, and Google Drive for Desktop to keep every team member's files live-synced. This guide walks through every step from provisioning to go-live.

Phase 1

Google Workspace
Foundation Setup

Configure the Workspace accounts that will anchor every Claude Pro seat. Done once. Takes ~45 minutes. Requires Super Admin access to the @thewideawakening.com Workspace console.

Step-by-Step

  • 01
    Log into Google Workspace Admin Console
    Go to admin.google.com. Sign in with your Super Admin @thewideawakening.com account. Verify you see the full admin dashboard — not just basic settings.
  • 02
    Audit existing user accounts
    Navigate to Directory → Users. Confirm which team members already have @thewideawakening.com accounts and which need to be created. You need accounts for: Jennifer, Jeannie, Admin Assistant, Program Manager, Video Editor, and Social Media Manager.
  • 03
    Create any missing user accounts
    Click Add new user for each missing team member. Recommended naming convention:
    firstname@thewideawakening.com — or for roles —
    programs@thewideawakening.com, socialmedia@thewideawakening.com, admin@thewideawakening.com, video@thewideawakening.com
  • 04
    Assign Jeannie as secondary admin (if not already)
    In Admin roles, assign Jeannie's account Super Admin or at minimum Drive Admin privileges. She'll need this to manage the Shared Drive and Sync Protocol independently of Adam.
  • 05
    Verify Google Drive is enabled for all users
    Go to Apps → Google Workspace → Drive and Docs. Confirm Drive is ON for all users in your org. Also confirm Shared Drives are enabled — this is a separate toggle under Drive settings.
  • 06
    Enable external sharing for the AI Outputs folder
    In Drive → Sharing settings, confirm that files can be shared with people outside the organization. This is needed so Claude outputs can be shared with external collaborators or clients when needed. Set to "Anyone with link can view" as the outer limit — actual sharing is controlled at the folder level.
  • 07
    Confirm all accounts can access Drive for Desktop
    In Drive → Drive for Desktop, confirm the app is allowed for your org. This is the setting that lets team members install the desktop sync client. If it shows "Blocked," toggle it to allowed before proceeding to Phase 4.
  • 08
    Send welcome/credentials to any new accounts
    Google Workspace will auto-generate a welcome email. Ensure each new team member receives their temporary password and knows to sign in at workspace.google.com. Ask them to change their password before Phase 3.
Note on Claude Pro Billing Claude Pro is $20/month per seat (as of June 2026). Each team member will have their own Claude Pro account registered to their @thewideawakening.com email. This keeps accounts as organizational assets. Budget: 5 seats × $20 = $100/month. Consider whether Jennifer needs a separate account or shares with leadership.

Phase 2

Drive Architecture &
Document Deployment

Create the Shared Drive

  • 01
    Create a new Shared Drive named "TWA Claude OS"
    In Google Drive, click Shared drives in the left sidebar → New. Name it TWA Claude OS. This is the organizational root for everything Claude-related.
  • 02
    Add all team members with appropriate permissions
    Right-click the Shared Drive → Manage members. Add each team member:
    jennifer@ → Content Manager
    jeannie@ → Manager (full edit + admin)
    admin@ → Contributor
    programs@ → Contributor
    video@ → Contributor
    socialmedia@ → Contributor

Upload MASTER Documents

  • 03
    Upload all six MASTER documents to _AI Operations/
    These already exist from prior work sessions. Copy or move them into the Shared Drive _AI Operations/ folder. If they're currently in My Drive, drag them in — or use Move to rather than copy, so there's only one authoritative copy.
  • 04
    Upload the six Avatar Research files to _AI Operations/Avatar Research/
    These are the evidentiary research files (briefing.md, audience-segments.md, etc.). They do not go into Claude Project Knowledge — they're reference material for Jeannie and Jennifer when updating the MASTER Avatar Profile.
  • 05
    Upload HTML reference documents to _Team Reference/
    Move the Prompt Playbook Reference, Sync Protocol, Team Rollout Guide, and System Map into _Team Reference/. These are read-only reference tools for the team — not loaded into Claude.
  • 06
    Set _AI Operations/ permissions to Viewer for non-admin team members
    Right-click _AI Operations/ → Share. Confirm that Admin Assistant, Program Manager, Video Editor, and Social Media Manager have Viewer access — not Editor. Only Jeannie and Jennifer should be able to modify MASTER documents.
  • 07
    Verify each team member can access their _AI Outputs/[Role]/ folder
    Share each role's output folder with that team member at Contributor level (can add files, can't delete others' files). Jeannie gets Manager access to all output folders — she's the review gate.
Important — One Source of Truth Once MASTER documents are in the Shared Drive, any copies in personal My Drive folders should be deleted or converted to shortcuts. The Shared Drive version is the only version the team should ever reference or sync from.

Phase 3

Claude Pro Accounts
& Project Configuration

Create Claude Pro accounts for each team member, build their role-specific Projects, and load Project Knowledge. This is where the AI layer comes to life. Budget ~60 minutes — 10–15 minutes per team member.

Account Creation (per team member)

  • 01
    Each team member creates a Claude account at claude.ai
    Go to claude.ai → Sign up. Use their @thewideawakening.com Google Workspace email. Click Continue with Google — this keeps account creation frictionless and ties the Claude account to the Workspace identity.
  • 02
    Upgrade each account to Claude Pro
    In Claude → click account name → Upgrade to Pro → $20/month. Use a TWA credit card or expense each seat. Confirm billing name is the team member's Workspace account name — not a personal name — for organizational clarity.
  • 03
    Confirm access to the Projects feature
    In the Claude sidebar, confirm Projects appears. If it doesn't show immediately after Pro upgrade, wait 5 minutes and refresh. Projects is a Pro-only feature — it won't appear on free accounts.

Project Setup (per team member)

  • 04
    Create a new Project named "TWA — [Role Name]"
    Click Projects → New Project. Name it using the convention:
    TWA — Jennifer Hough
    TWA — GM · Jeannie
    TWA — Social Media
    TWA — Program Manager
    TWA — Admin Assistant
    TWA — Video Editor
  • 05
    Paste the Project Instructions (system prompt)
    In Project Settings → Project Instructions, paste the full TWA Claude OS system prompt (the shared Project Instructions document). This is the same base prompt for all team members. The role-specific customization comes from Project Knowledge, not the instructions — so all 6 Projects share the same instructions.
  • 06
    Add the six MASTER documents as Project Knowledge
    In Project Settings → Project Knowledge → Add Content → Paste Text. Open each MASTER document from the Shared Drive (via Drive for Desktop or browser), copy the full text, and paste into Claude. Add one document at a time. Name each piece exactly as the document is named in Drive: MASTER — Brand Voice, etc.
  • 07
    Add the role-specific Prompt Playbook section
    From the TWA Prompt Playbook Reference, copy the section for this team member's role and paste it as an additional Project Knowledge item named Prompt Playbook — [Role]. This gives them quick-reference prompts without needing to open the HTML document.
  • 08
    Run the verification prompt
    Start a new conversation in the Project. Type: "Summarize TWA's current offer ecosystem and tell me who our ideal client is." The response should accurately reflect both the Offer Library and Avatar Profile. If it gives generic answers, the Project Knowledge didn't load correctly — return to Step 6.
Project Knowledge Limit Claude Pro Projects support up to approximately 200,000 tokens of Project Knowledge (roughly 150,000 words). The six MASTER documents plus a role Playbook should use well under half that. You have significant room to add more reference material as the system evolves.

Phase 4

Team Activation
& Go-Live

Install Drive for Desktop on all machines, run team orientation, execute verification prompts across all roles, and hand off operational ownership to Jeannie. This is the final phase before the Claude OS is live and self-sustaining.

Drive for Desktop Installation (every machine)

  • 01
    Download and install Google Drive for Desktop
    Each team member goes to drive.google.com/drive/downloads and downloads the installer for their OS (Mac or Windows). Install takes ~5 minutes. Sign in with @thewideawakening.com account when prompted.
  • 02
    Add the "TWA Claude OS" Shared Drive to synced folders
    In Drive for Desktop preferences → Google Drive tab → confirm Shared drives are set to sync. The TWA Claude OS Shared Drive should appear and begin syncing. On Mac it appears under Locations in Finder. On Windows it appears as a mapped drive letter (typically G:\).
  • 03
    Confirm the _AI Operations folder is accessible locally
    Each team member should be able to navigate to TWA Claude OS → _AI Operations from their file browser and open a MASTER document without a browser. This confirms sync is working. If the folder is empty, wait 2–3 minutes for initial sync to complete.

Team Orientation

  • 04
    Schedule a 30-minute team orientation call
    Jeannie leads. Agenda: (1) show the Drive structure, (2) demo opening a MASTER doc from Drive for Desktop, (3) demo running a prompt in Claude Project, (4) show where outputs go, (5) explain the Sync Protocol. Record the call for team members in different time zones.
  • 05
    Each team member runs their first real-world prompt
    Not a test prompt — an actual task from their role. Social Media Manager: draft a caption for an upcoming Awakened Conversations event. Program Manager: outline the onboarding sequence for a new Flight School participant. This locks in the workflow muscle memory.
  • 06
    Each team member saves their first output to the correct Drive folder
    After running their first prompt, team members copy the output, create a new Google Doc in their _AI Outputs/[Role]/Drafts/ folder, paste the content, and name it using the Output Naming Convention. Jeannie spot-checks naming compliance.
  • 07
    Jeannie reviews first outputs from each team member
    Jeannie reviews all Drafts from the orientation session. Anything that passes her quality and voice check gets moved to Approved/. Anything that needs revision goes back with a comment in the Google Doc. This establishes the review pipeline in practice, not just theory.
  • 08
    Adam hands off to Jeannie as system owner
    Adam's role shifts from builder to advisor. Jeannie takes ownership of: the Sync Protocol, new team member onboarding, and the monthly sync check. Adam remains available for architecture questions, upgrades, and Phase 2 (Supabase) planning. Formalize this in a brief handoff document.
